PET ADOPTION & ANIMAL WELFARE - PROJECT REPORT
================================================================================

1. PROJECT OVERVIEW
-------------------
This project is a comprehensive web platform dedicated to promoting pet adoption 
and animal welfare awareness. It serves as a bridge between loving homes and 
animals in need, while providing educational resources on responsible pet 
ownership.

2. TECHNOLOGIES USED & WHY
--------------------------
* HTML5: Used for the core structure and semantic organization of web content.
* CSS3: Used for styling, layout (Flexbox/Grid), and premium animations/effects.
* JavaScript: Used for client-side logic, interactivity, and integration.
* Firebase (Auth/Firestore): Used for backend services like user authentication 
  and real-time data management.
* Netlify Forms: Used for serverless contact form handling and message storage.
* Swiper.js: A modern library used to create responsive, high-quality image 
  sliders/galleries.
* Font Awesome: Used for professional vector icons throughout the site.
* Google Fonts (Roboto): Used for modern, readable typography.

3. TOPICS & FEATURES DETAILS
----------------------------

A. NAVIGATION & HEADER
   - Where: Top of every page.
   - Logic: A fixed header that remains accessible during scrolling. It contains 
     a responsive navbar that transforms into a hamburger menu on mobile.
   - Working: Uses CSS 'position: fixed' and JS toggle logic for the menu.

B. AUTHENTICATION SYSTEM (LOGIN/SIGNUP)
   - Why: To allow users to create accounts, save preferences, and interact 
     with protected sections.
   - Frontend Working: A dropdown modal ('#auth-modal') that toggles between 
     Login, Signup, and Reset Password forms without page reloads.
   - Backend Working: Uses Firebase Authentication (v10 COMPAT).
   - Logic:
     * Signup: Captures Name, Email, and Password. Creates user in Firebase 
       and updates the profile with the display name.
     * Login: Validates credentials against Firebase Auth.
     * Password Reset: Sends an automated email link via Firebase.
     * Persistence: 'onAuthStateChanged' observer updates the UI (showing name, 
       logout button) based on the user's login state.

C. HOME / HERO SECTION
   - Why: To grab user attention and provide a clear call-to-action (CTA).
   - Logic: Beautiful background image with high-contrast text and a wave 
     animation ('bottom_wave.png') to transition into the next section.

D. ABOUT & BANNER SECTIONS
   - Why: To tell the story and mission of the animal welfare initiative.
   - Logic: Uses 'deco-title' for stylised background text and responsive 
     flex-containers for text/image layouts.

E. GALLERY (SWIPER.JS)
   - Why: To showcase animals in a visually appealing, interactive way.
   - Logic: Swiper.js initialization in 'script.js' with 'loop: true', 
     'centeredSlides', and custom navigation arrows.
   - Working: Supports touch-swipe on mobile and arrow navigation on desktop.

F. ANIMAL WELFARE (SERVICE CARDS)
   - Where: Middle section of the page.
   - Logic: A grid-based layout of animal cards.
   - Working: Uses CSS hover effects to show a dark overlay and a "See Details" 
     button with smooth transitions.

G. CONTACT FORM (BACKEND INTEGRATION)
   - Why: To allow visitors to reach out for inquiries.
   - Frontend Working: Standard HTML form with validation (required fields).
   - Backend Working: Integrated with Netlify Forms using 'data-netlify="true"'.
   - Logic:
     * No-Code Backend: Netlify captures the POST request automatically.
     * Security: Includes a 'bot-field' honeypot to prevent spam.
     * Data: captures Name, Email, Number, Subject, and Message.

H. FOOTER & NEWSLETTER
   - Logic: Contains quick links, contact info, and a newsletter subscription.
   - Working: Subscription alert is handled via JS (placeholder logic).

4. FULL WORKING LOGIC (FRONTEND & BACKEND)
------------------------------------------
FRONTEND:
The frontend is built using standard HTML/CSS/JS. Responsiveness is handled 
via CSS Media Queries for breakpoints at 991px, 768px, and 450px. Interactivity 
(modals, menu toggles, scrolling behavior) is managed by 'script.js'.

BACKEND:
- Authentication: Handled entirely by Firebase Client SDK.
- Form Submissions: Handled by Netlify's built-in forms processing.
- Security: A Content Security Policy (CSP) meta tag in HTML ensures only 
  trusted external scripts (Firebase, Swiper, Google Fonts) can load.

5. LOGIN & CONTACT FORM DETAILS
-------------------------------
LOGIN FORM:
- Located in an absolute-positioned div that drops down from the user icon.
- Toggles 'active' class via JS.
- Uses Firebase 'signInWithEmailAndPassword' and 'createUserWithEmailAndPassword'.

CONTACT FORM:
- Uses the 'POST' method to send data to the same URL.
- Netlify intercepts the submission because of the 'data-netlify' attribute.
- Fields: name, email, number, subject, message.
